ingui是static的，在hazel这个dll中可以被很好的使用。但是在exe文件中会链接出错，这是为什么？我们的sandbox只链接hazel，而hazel是dll，他只链接使用过的静态库的符号，并且在imgui.h修改export符号的宏IMGUI_API编译才可以。 我们可以hack一下。在imgui预编译输入IMGUI_API=_declspec(dllexport),然后再hazel定义IMGUI_API=_declspec(dllexport)然后再使用相关的符号来import符号到dll来 还可以定义hazel的linker的module definition file的.def文件来import所有的符号， 所以我们选择把hazel编译成静态库。因为我们不太需要动态库

build 2D renderer

我们好像已经有了一个renderer的实际的代码了。我们怎么把这些东西抽象起来？

我们的2d renderer不太需要一些submit。我们并不过多的需要顶点。我们可能只需要纹理加上一个四边形。所以我们的Renderer::Submit()就可以替换成Renderer::DrawQuad()

这里引申出了我们需要batch rendering的技术来实现很多四边形的绘制。我们一个一个的draw call就不合适了。通过bacth的技术把所有的draw call化为一个draw call。

流程： - Renderer::BeginScene(camera) - Renderer::DrawQuad(pos,shader,color,texture) - batch renderer - animation - ui-layout system 我们要适应不同的屏幕比例，resize等 - post effect 粒子效果等 - interaction ：scrpting，在ECS系统很有用，因为一个object不是通过继承组成，而是component。比如一个player entity由transfrom component，2D renderer component，input component（hook key input）等。
